# Sentence Library Generation

The tailored cloze sentences shown in Practice (Sentence mode) come from a public
Supabase table, `public.sentences`, keyed by `(word_key, type)`. This doc is the
operational runbook for filling that table.

The pipeline makes **no LLM API calls**. Sentences are generated by **Codex**
(run manually on your machine via the ChatGPT Pro plan); the scripts only emit
work and validate/import the results. When the table has no row for a card, the
app falls back to the offline generator, so seeding is incremental and safe.

## How it fits together

1. `npm run sentences:batches` writes JSONL batch files of pending
   `(word, conjugation)` pairs, plus a `PROMPT.md`.
2. **Codex** reads each batch and writes a matching output JSONL of sentences.
3. `npm run sentences:import <output.jsonl>` validates each sentence against the
   conjugation engine, derives per-token furigana with **kuromoji**, and upserts
   to Supabase. Rejected lines go to `<output>.rejects.jsonl`.

The conjugated form is recomputed from the engine and must appear verbatim in the
sentence, so a wrong tense/form is rejected rather than stored.

## One-time setup

Install deps and set the Supabase service credentials (the **`service_role`**
key from Supabase → Project Settings → API — not the anon key):

```bash
npm install
export SUPABASE_URL="https://<your-project-ref>.supabase.co"
export SUPABASE_SERVICE_ROLE_KEY="<service_role key>"
```

Optionally regenerate the lexicon so offline fallbacks get real JMdict
transitivity (needs network), and commit the result:

```bash
npm run vocab:build
git add public/data/verb-lexicon.json
git commit -m "Regenerate verb lexicon with transitivity"
```

## Environment variables

| Var | Default | Meaning |
| --- | --- | --- |
| `SENTENCE_JLPT` | all levels | Comma list of JLPT levels to include, e.g. `N5,N4`. |
| `SENTENCE_TYPES` | **all 126 forms** | Comma list of type ids, `all`, or `default` (the 28 learner-default forms). |
| `SENTENCE_BATCH_SIZE` | `200` | Pairs per batch file. |
| `SENTENCE_LIMIT` | `0` (no cap) | Cap total pairs emitted (for smoke tests). |
| `SENTENCE_OUT_DIR` | `tmp/sentence-batches` | Where batches/outputs are written (gitignored). |
| `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` | — | Required to import; when set, `sentences:batches` skips pairs already in the DB (resumable). |
| `SENTENCE_DRY_RUN` | unset | `1` validates without upserting. |
| `SENTENCE_NO_KUROMOJI` | unset | `1` uses model-provided segments instead of kuromoji. |

`sentences:batches` now defaults to **every form for every word at every JLPT
level** — a plain run covers everything the engine can produce. Narrow it with
`SENTENCE_JLPT` / `SENTENCE_TYPES` when you want a smaller tranche.

## Run everything (instructions for Codex)

Paste the following to Codex. With `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY`
set, it will work through the entire lexicon, resuming where it left off.

> **Task: fill the tailored sentence library for every word and form.**
>
> The repo has a generation pipeline; you (Codex) are the generator. The scripts
> make no LLM calls. `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` are set.
>
> 1. `git checkout main && git pull && npm install`
> 2. Emit batches for everything: `npm run sentences:batches`
>    (every word × every applicable form, skipping pairs already in the DB).
>    This writes `tmp/sentence-batches/PROMPT.md` and `batch-0001.jsonl`, …
> 3. Read `tmp/sentence-batches/PROMPT.md`. Then process **every** `batch-*.jsonl`
>    file in order. For each batch, write a matching `out-*.jsonl` where each line
>    is `{ "word_key", "type", "ja", "en" }`:
>    - `ja` is one short, natural, beginner-appropriate sentence that uses the
>      word in the requested form and contains `expected_surface` **verbatim,
>      exactly once**.
>    - Use the `transitive` hint and `type_label` to phrase the form correctly.
>    - Do not include readings; furigana is derived automatically. JSONL only.
> 4. Validate: `SENTENCE_DRY_RUN=1 node scripts/import-sentence-library.js tmp/sentence-batches/out-XXXX.jsonl`
>    Fix any lines listed in `out-XXXX.rejects.jsonl` (usually the sentence
>    didn't contain `expected_surface`) and re-validate until clean.
> 5. Import: `node scripts/import-sentence-library.js tmp/sentence-batches/out-XXXX.jsonl`
>    (idempotent upsert by `(word_key, type)`).
> 6. Continue to the next batch file until all are imported. If interrupted,
>    re-run `npm run sentences:batches` — it regenerates only the pairs still
>    missing from the DB.
>
> Notes: `tmp/` is gitignored — do not commit batch/output files. The only thing
> worth committing is a regenerated `public/data/verb-lexicon.json` from
> `npm run vocab:build`.

## Checking progress

```sql
select count(*) from public.sentences;
select type, count(*) from public.sentences group by type order by 2 desc;
```

Imported sentences appear in Practice → Sentence mode automatically for the
covered words. Full lexicon × all forms is ~150k–270k sentences, so expect a long
run; value lands continuously as batches import (no redeploy needed).
